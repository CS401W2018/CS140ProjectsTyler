1) The form tag has an "action" attribute. What does it do?
The action attribute is equivalent to the URl that you are sent to when you submit the form. So for this web page its process.html which doesn't exist which is why it gets mad at your when you try to submit the form.

2)The form tag has a "method" attribute. What does it do?
The method attribute tells the web page how to send the data within the form. It can use either a GET or a POST request to send the data.

3)What is the difference between a "POST" and a "GET"?
The difference between a POST and a GET request is that POST requests are used for sending data and GET requests are used for reading data. When you want to submit and make a change on the server, you would use a POST request, one small difference is the information you send is not available in the URl in a POST request but that doesn't make it more secure if your on HTTP, using something like wireshark you would still be able to see the data being passed. This is why you use HTTPS which encrypts the data in motion between the 2 computers. GET requests though are mainly used though for when you want to read data and your not making a change within the server.
